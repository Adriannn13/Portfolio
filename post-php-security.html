Deep Dive: PHP & MySQL Security Logic

"Trust no user input."

What I Did

While working on my OJT Monitoring System, I realized that just because an app works doesn’t mean it’s secure. So instead of using standard queries, I switched to prepared statements using PDO throughout the system. This helps prevent SQL injection by keeping the query structure separate from the actual data being entered.

$stmt = $pdo->prepare('SELECT * FROM users WHERE email = :email');
$stmt->execute(['email' => $userEmail]);
$user = $stmt->fetch();

Example of my PDO implementation

I also focused on credential safety. Instead of storing plain text (a huge red flag), I utilized password_hash() with the BCRYPT algorithm. For the admin dashboard, I built a custom session-checking middleware that validates user roles before rendering any sensitive data.

Key Insight

Backend development is 20% building features and 80% protecting them. Learning to think like a "breaker" helped me become a much better "maker." If you don't sanitize your inputs, you're essentially leaving your front door wide open.

The Result

By focusing on security from the very beginning, the OJT Portal became more than just a school project—it turned into a system people can trust. It taught me that as a developer, my main responsibility is to protect the user’s data and keep it safe.
