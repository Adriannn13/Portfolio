Transitioning from Student to Junior Developer

"Learning how to learn is the ultimate skill."

What I Did

The biggest shift in my journey wasn't learning a new language—it was changing my workflow. I moved away from "tutorial hell" and started building real-world projects like my Car Marketplace Capstone. I began using Postman to test API endpoints before touching the frontend, and I started relying on official documentation instead of just stack overflow.

I also embraced version control (Git) more seriously, writing descriptive commit messages and branching for new features. This organized approach allowed me to handle larger projects without getting lost in my own code.

Key Insight

The biggest hurdle in coding isn't the syntax—it's the logic. Once I realized that code is just a tool to solve human problems (like tracking OJT hours or calculating profit margins), the technical side became much easier to grasp.

The Future

Being a Junior Dev means being a professional student. I’m currently diving deeper into Laravel and exploring React.js integration to build even faster, more reactive web applications. The journey is long, but every bug solved is a step forward.
